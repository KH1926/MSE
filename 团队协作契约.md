| 字段 | 类型 | 说明 |
| --- | --- | --- |

## 5. API 接口规范（A 定义，B/C 调用）

### 5.1 统一响应格式

所有接口均返回以下结构：

```json
{
  "code": 200,
  "message": "ok",
  "data": {}
}
```

错误时 `code` 为非 200，`message` 为错误说明，`data` 为 `null`。

### 5.2 核心接口列表

| 方法 | 路径 | 说明 | 消费方 |
| --- | --- | --- | --- |
| `POST` | `/orders` | 新建销售订单 | A 实现，C 调用 |
| `GET` | `/orders` | 查询订单列表 | A 实现，C 调用 |
| `POST` | `/orders/{order_no}/generate-work-order` | 订单转工单 | A 实现，B/C 调用 |
| `POST` | `/schedule` | 触发排产 | A 实现，B 提供逻辑，C 调用 |
| `GET` | `/schedule/tasks` | 查询排产任务（甘特图数据源） | A 实现，C 调用 |
| `POST` | `/simulation/run` | 触发仿真（传入场景参数） | A 实现，B 提供仿真逻辑 |
| `GET` | `/stations/status` | 查询所有工位当前状态 | A 实现，C 调用 |
| `GET` | `/events` | 查询生产事件日志 | A 实现，C 调用 |
| `GET` | `/quality/records` | 查询质检记录 | A 实现，C 调用 |

### 5.3 仿真触发接口参数（B 定义入参，A 实现接口）

```json

```

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `scenario` | string | 场景编码，见第 4 节 |

## 7. KPI 计算口径（B 定义，A 落库，C 展示）

| 指标 | 字段名 | 计算口径 |
| --- | --- | --- |
| 总完工量 | `total_completed` | 仿真周期内 `work_order_status = completed` 的工单数 |
| 订单准时率 | `on_time_rate` | `actual_end <= due_date` 的订单数 ÷ 总完工订单数 |
| 平均生产周期 | `avg_lead_time` | 所有完工工单的 `(actual_end - actual_start)` 均值（分钟） |
| 在制品数量 | `wip_count` | 仿真结束时 `work_order_status = in_progress` 的工单数 |
| 不良率 | `defect_rate` | `quality_result != pass` 的质检记录数 ÷ 总质检数 |
| 返修率 | `repair_rate` | `quality_result = repair` 的质检记录数 ÷ 总质检数 |
| 生产线平衡率 | `line_balance_rate` | 各工位基准时间 `mode_time` 之和 ÷ (工位数 × 最大 `mode_time`) |
| 瓶颈工位 | `bottleneck_station` | 基准时间 `mode_time` 最大的工位 ID（固定为 `WS20`） |

## 8. 时间单位约定（全员共用）

- 所有**工位随机分布参数**（`min_time`、`mode_time`、`max_time`、`sigma`）：以**秒**为单位
- 所有**仿真内部时间**：以**分钟**为单位（SimPy 时间轴单位 = 1 分钟）；工位随机分布参数在仿真执行前统一由秒换算为分钟
- 所有**数据库存储时间**：使用 `YYYY-MM-DD HH:MM:SS` 字符串格式（SQLite TEXT 类型）
- 所有**前端展示时间**：工位参数展示统一使用 `s`，聚合结果如平均生产周期可显示“X 分钟”或“X.X 小时”
- 工位随机分布参数表中的时间字段：以**秒**为单位
