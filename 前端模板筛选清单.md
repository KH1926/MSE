# 前端模板筛选清单

## 1. 结论先行

本项目不建议从零开发前端，建议直接复用现成后台模板。

推荐决策规则：

- 队内有人能完成基础前端改造与接口对接：选 `Vue3 Dashboard Template + ECharts + FastAPI`
- 队内前端能力较弱，或 `2` 天内模板仍未跑通：立刻切换到 `FastAPI + Jinja2 + Bootstrap Admin Template`

核心原则：

- 模板复用优先于从零开发
- 算法、仿真、实验优先于页面美化
- 页面目标是“稳定展示结果”，不是“自研后台系统”

## 2. 选择标准

筛选模板时只看以下几点：

1. 能否快速启动并跑通
2. 是否自带后台布局、表格、图表和菜单
3. 是否容易替换成订单、排产、监控、质检这 `5` 个页面
4. 是否便于接 `FastAPI` 接口
5. 是否不要求大量自定义样式开发

## 3. 推荐路线

### 路线 A：模板化前后端分离

适用情况：

- 队内至少 `1` 人会基本 `Vue`
- 能处理接口请求、表格渲染和图表绑定

建议用途：

- 首页驾驶舱
- 订单管理页
- 排产中心页
- 生产监控页
- 质量分析页

### 路线 B：模板化服务端渲染

适用情况：

- 前端能力弱
- 时间紧
- 目标以稳定演示为主

建议用途：

- 通过 `Jinja2` 直接渲染表格和图表数据
- 用现成 `Bootstrap` 后台模板改字段和卡片
- 只保留最必要的交互

## 4. 推荐模板清单

以下模板基于 `2026-04-14` 对 GitHub 项目的简要筛选，适合作为本项目候选。

### 4.1 Vue3 路线候选

#### 候选 1：`v3-admin-vite`

- 链接：[un-pany/v3-admin-vite](https://github.com/un-pany/v3-admin-vite)
- 推荐级别：`A`
- 优点：
  - `Vue3 + Vite + TypeScript + Element Plus` 组合成熟
  - 后台布局完整，适合直接改成比赛页面
  - 社区活跃，文档和示例较全
- 风险：
  - 功能较全，初次接触时需要删减无关模块
- 适合：
  - 需要完整后台框架且队内有人能改 `Vue3`

#### 候选 2：`vue-element-plus-admin`

- 链接：[kailong321200875/vue-element-plus-admin](https://github.com/kailong321200875/vue-element-plus-admin)
- 推荐级别：`A-`
- 优点：
  - `Vue3 + Vite + TypeScript + Element Plus`
  - 常用后台页面齐全，接口对接思路清晰
  - 适合快速改出订单、排产、监控页面
- 风险：
  - 结构较完整，仍需精简模块
- 适合：
  - 想要“开箱即用”程度更高的后台模板

#### 候选 3：`art-design-pro`

- 链接：[Daymychen/art-design-pro](https://github.com/Daymychen/art-design-pro)
- 推荐级别：`B+`
- 优点：
  - 视觉效果较好，适合做展示型页面
  - `Vue3 + Vite + TypeScript + Element Plus`
- 风险：
  - 更偏视觉体验，不一定是最快的落地选项
- 适合：
  - 已有人负责前端展示，并且想提升答辩观感

### 4.2 Bootstrap 路线候选

#### 候选 1：`AdminLTE`

- 链接：[ColorlibHQ/AdminLTE](https://github.com/ColorlibHQ/AdminLTE)
- 推荐级别：`A`
- 优点：
  - 经典后台模板，页面结构成熟
  - 表格、卡片、菜单、图表区域齐全
  - 改字段和替换图表速度快
- 风险：
  - 默认页面较多，需要删掉无关内容
- 适合：
  - 用 `FastAPI + Jinja2` 快速出一个稳定演示系统

#### 候选 2：`PlainAdmin`

- 链接：[PlainAdmin/plain-free-bootstrap-admin-template](https://github.com/PlainAdmin/plain-free-bootstrap-admin-template)
- 推荐级别：`A-`
- 优点：
  - 组件比较直接，适合快速替换成业务页面
  - 对前端基础要求低
- 风险：
  - 生态和资料不如 `AdminLTE` 广
- 适合：
  - 希望界面更简洁，减少额外样式工作

#### 候选 3：`AdminKit`

- 链接：[adminkit/adminkit](https://github.com/adminkit/adminkit)
- 推荐级别：`B+`
- 优点：
  - 页面整洁，后台风格明确
  - 适合快速拼装驾驶舱和业务页
- 风险：
  - 某些构建细节需要熟悉其工程结构
- 适合：
  - 想要简洁风格，同时保留较完整后台布局

## 5. 最终推荐

### 推荐 1：前端能力中等

直接选：

- `v3-admin-vite + FastAPI`

理由：

- 页面框架成熟
- 改造成 `5` 个核心页面成本较低
- 看板、表格、菜单、卡片都容易拼装

### 推荐 2：前端能力偏弱

直接选：

- `AdminLTE + FastAPI + Jinja2`

理由：

- 不需要完整前后端分离
- 后端直接渲染数据，联调成本低
- 更适合在有限时间内保住演示闭环

## 6. 页面改造范围

不管选哪条路线，页面只做以下 `5` 个：

1. 首页驾驶舱
2. 订单管理页
3. 排产中心页
4. 生产监控页
5. 质量分析页

只允许改：

- 菜单名称
- 表格字段
- 图表数据绑定
- 少量卡片文案

不建议投入时间在：

- 自定义复杂动画
- 重写整体样式系统
- 做权限、登录、主题切换等无关功能

## 7. 快速试用流程

### Day 1（对应项目 D3）：模板选定

- 根据队内已有前端能力，**直接选定一套模板**（Vue3 能力中等选 `v3-admin-vite`，偏弱选 `AdminLTE`）
- 不需要同时试跑两套，节省决策时间
- 本地跑通首页，确认能替换菜单和图表区域

### Day 2（对应项目 D4）：接口联调测试

- 接 `B` 输出的 `/mock` 静态 `JSON` 数据
- 跑通订单列表或首页 `KPI` 卡片渲染
- 判断改造复杂度；若仍卡壳，当天切换另一路线

### Day 3（对应项目 D5 起）：正式推进页面改造

- 模板已选定，不再反复更换
- 基于 `mock` 数据独立推进，不等真实接口

## 8. 使用建议

- 前端负责人只负责“模板改造与展示”
- 不额外承担排产算法或仿真主逻辑
- 模板一旦选定，不再反复更换
- 页面以可演示、可截图、可写文档为唯一标准

## 9. 一句话总结

本项目的前端策略应是：**直接复用模板，快速改成比赛页面，把时间留给排产、仿真、实验和文档。**
